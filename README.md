# SBOMEnrich

The SBOMSCAN is a free, open source tool to enrich or update a SBOM (Software Bill of Materials) in either [SPDX](https://www.spdx.org) and [CycloneDX](https://www.cyclonedx.org) format with additonal information.

The tool can enrich SBOM metadata or SBOM package attribute data.

The following package attributes can be enriched:

| Atrribute| Description|
| --------- |---------|
| licenseconcluded | A valid SPDX license identifer |
| licensedeclared  | A valid SPDX license identifer |
| supplier | The component supplier |
| copyright | A copyright statement      |
| type | Component type e.g. library, file |
| downloadlocation | A valid URL specifiying the homepage for the component |
| property | A user defined attribute |
| purl | A valid PURL|
| checksum | A SHA-512 hash value |

Note: property attributes consist of two components, the name and the value. The format of the enriched data is <name>#<value> e.g. dataclass#PII where dataclass is the name of the property and PII is the value.

## Installation

To install use the following command:

`pip install sbomenrich`

Alternatively, just clone the repo and install dependencies using the following command:

`pip install -U -r requirements.txt`

The tool requires Python 3 (3.10+). It is recommended to use a virtual python environment especially
if you are using different versions of python. `virtualenv` is a tool for setting up virtual python environments which
allows you to have all the dependencies for the tool set up in a single environment, or have different environments set
up for testing using different versions of Python.

## Usage

```bash
usage: sbomenrich [-h] [-i INPUT_FILE] [--enrich ENRICH] [-d] [--sbom {spdx,cyclonedx}] [--format {tag,json,yaml}] [-o OUTPUT_FILE] [-V]

Sbomenrich updates a Software Bill of Materials by amending the attributes for components

options:
  -h, --help            show this help message and exit
  -V, --version         show program's version number and exit

Input:
  -i INPUT_FILE, --input-file INPUT_FILE
                        filename of SBOM
  --enrich ENRICH       filename containing enrichment data

Output:
  -d, --debug           add debug information
  --sbom {spdx,cyclonedx}
                        specify type of sbom to generate (default: cyclonedx)
  --format {tag,json,yaml}
                        specify format of software bill of materials (sbom) (default: json)
  -o OUTPUT_FILE, --output-file OUTPUT_FILE
                        output filename (default: output to stdout)

```

## Operation

The `--input-file` option is used to identify the SBOM to be enriched. Both SPDX and CycloneDX SBOMs are supported. The `--enrich-file` option is used to identify the enriched data. Both the `--input-file` and `--enrich-file` options must be provided.

The `--sbom` option is used to specify the format of the generated SBOM (the default is SPDX). The `--format` option
can be used to specify the formatting of the SBOM (the default is Tag Value format for a SPDX SBOM). JSON format is supported for both
SPDX and CycloneDX SBOMs).

The `--output-file` option is used to control the destination of the output generated by the tool. The
default is to report to the console but can be stored in a file (specified using `--output-file` option).

## Enrichment Example

The following enrichment file

```yaml
# Sample enrichment file
packages:
- name: click
  licenseconcluded: MIT
  property: dataclass#PII
- name: requests
  licenseconcluded: MIT
```

## Limitations

This tool only processes valid SBOMs.

The tool does not validate that the enrichment data is valid for the specified attribute. Invalid values will be siliently ignored.
